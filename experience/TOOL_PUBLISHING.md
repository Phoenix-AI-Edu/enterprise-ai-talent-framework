# AI 工具上架與發布機制

本機制讓每個新工具以相同方式進入鳳凰 AI 官網，避免行銷說法、Demo 現況與實際產品脫節。

## 責任分工

| 角色 | 責任 |
| --- | --- |
| 官網整合負責人 | 維護資料模型、頁面、連結、分析事件與發布品質。 |
| 工具負責人 | 提交功能證據、Demo 網址或影片、限制與上次驗證日期。 |
| 商業內容審核人 | 核准定位、受眾、CTA 與 PoC 成功標準。 |
| 發布核准人 | 核准首頁曝光與正式發布。 |

工具負責人不得直接將未驗證功能寫入首頁或案例區；所有修改透過 Pull Request 由官網整合負責人合併。

## 提交流程

1. 以 `tool-submission-template.json` 填寫工具資料。
2. 工具負責人提供每項「現可操作」功能的證據：公開 Demo、錄影或可重現步驟。
3. 商業內容審核人確認定位、CTA 和不做什麼。
4. 官網整合負責人建立卡片與詳情頁，並執行 `npm run validate:tools`、連結與手機版檢查。
5. 發布核准人確認後才合併至 `main`。

## 能力標示規則

- `available_now`：訪客可在公開 Demo 或錄影中驗證的能力。
- `poc_configurable`：在限定範圍、資料與人力配置後可驗證的能力。
- `implementation_assessment`：需另案評估的正式整合或客製能力。

任何未落在 `available_now` 的能力，不得寫成已上線、即時可用或客戶成效。

## 發布前檢核

- [ ] 產品名稱、狀態、CTA 在首頁、體驗區與詳情頁一致。
- [ ] 沒有未證實的客戶名稱、成效數字或正式系統串接宣稱。
- [ ] 所有內部連結可在 GitHub Pages 專案路徑下開啟。
- [ ] 主 CTA 已有 GA4/GTM 事件，且事件沒有個資欄位。
- [ ] 已在手機與桌面寬度檢查。
- [ ] `last_verified` 在 90 天內；過期工具要標示待更新或下架。
