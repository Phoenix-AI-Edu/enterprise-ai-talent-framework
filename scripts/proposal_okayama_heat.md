### 一、邊緣離線樹莓派與 10ms 實體繼電器硬熔斷架構（回應您第 1、2 項要求）

為了在老舊電爐與斷網環境下確保 100% 安全，陳策略長為您規劃了「地端邊緣離線推論 + 物理熔斷保護」架構：

```
   [熱處理熱電偶 / 碳勢探針] 
             │
             ▼ (Modbus RTU 唯讀)
   [本地工業級樹莓派 (Raspberry Pi 4)] ───► [LightGBM 爐溫碳勢預測模型 (地端離線)]
             │                                              │
             ▼ (實時推論溫度與閥門建議)                     ▼ (實時計算 AI 信心度)
   [PLC 控制器] ──(實體硬線)──► [物理繼電器 (Relay) 熔斷盒] ◄── [信心度判定閥值]
                                      │
                 🟢 信心度 ≥ 95% ─────┼──► [調節閥微調控制 (AI 模式)]
                 🔴 信心度 < 95% ─────┼──► [10ms 物理繼電器彈開，切斷 AI 寫入]
                                      ▼
                               [輝哥物理控制閥門 (手動模式)] ◄─── [輝哥手動一鍵否決實體按鈕]
```

#### 1.1 本地工業級樹莓派離線分析 API
我們完全反對將爐溫實時反饋數據上送雲端。我們將在熱處理爐旁的機電箱內部署一台免插網線、防高溫防磁的「工業級樹莓派微伺服器」，在本地地端運行輕量化的 `LightGBM` 與 `TimeGPT` 離線預測模型。
*   **非侵入式唯讀**：樹莓派僅透過 Modbus RTU 協議，以「唯讀」方式拉取爐內的溫度與碳勢傳感器數據，絕不直接修改 PLC 核心安全保護參數。
*   **零斷網死機風險**：即使廠房外網完全中斷，樹莓派依然能以 < 50ms 的超低延遲在本地進行預估，產出閥門微調建議，徹底規避雲端 API 重試造成的算法死機。

#### 1.2 10ms 實體硬繼電器雙軌熔斷器
我們在樹莓派與 PLC 控制器之間，加裝一個獨立的「物理繼電器安全控制盒」（Relay Cutoff Box）。
*   **信心度熔斷機制**：樹莓派在輸出調溫指令時，會同步計算當前物理特徵的「AI 信心度指數」。一旦發現數值漂移、電磁干擾或網路瞬斷，導致 AI 信心度低於 95%，繼電器線圈會瞬間失電，在 **10 毫秒（ms）內以物理彈開的方式切斷 AI 的控制信號通路**，系統無縫退回傳統的 PLC 安全保護與輝哥手動控制狀態。
*   **輝哥的一鍵物理否決按鈕（Veto Button）**：我們在熱處理爐旁加裝一顆直徑 10 公分的實體紅色「一鍵否決按鈕」，直接以硬體線路（Hardware Wire）連接繼電器控制電源。只要輝哥覺得火色不對，**按下去的瞬間，物理繼電器會直接斷電彈開，AI 寫入權限 100% 喪失**，系統主控權瞬間回到輝哥手中。這不是軟體畫面上的「停止」，是物理斷電，給輝哥最強大的物理掌控權。
