## 優化建議一回覆：與既有 ERP／CRM 的低成本整合（您的硬核要求）

1. **唯讀 API 串接（最低風險）：** AI Agent 以唯讀權限讀取 ERP/CRM 資料（如庫存、工單、客戶歷史），不寫回、不改動核心，CIO 零風險。

# 第二部分：實務落地技術解決方案與安控

> 本部分把第一部分的戰略，落成可施工的「架構 × 安控 × 流程」三層配置。所有設計恪守製造業鐵律：**簡單、耐操、不動既有系統核心、現場活得下來。** 我們不為一家精密製造廠堆砌金融業等級的重武裝，而是用「滿手油的師傅三秒會用、CIO 敢放行、CFO 算得出帳」的配置解決真實問題。

## 配置總綱：邊緣為主、輕量串接為輔的務實架構

製造業的架構現實是：①現場要即時、不能等雲端；②既有 ERP/CRM 動不得；③機密（客戶名單、財報、製程參數）不能外流。因此核心是**「邊緣優先（Edge-First）＋ 不動核心（Non-Invasive）」**：

| 資料／任務層級 | 場景範例 | 部署位置 | 理由 |
| :--- | :--- | :--- | :--- |
| **即時控制（紅區）** | Voice AI 語音報工報修、現場即時查詢 | **廠內邊緣節點** | 毫秒反應、產線不可斷、不依賴外網 |
| **機密數據（黃區）** | 客戶名單、財務報表、製程參數、工單 | **廠內伺服器／既有 ERP（唯讀串接）** | 商業機密與核心系統，不外流、不更動 |
| **彈性運算（綠區）** | 通用語音底座、模型微調、非敏感查詢 | **去識別化混合雲／租用 API** | 算力彈性、成本最優，僅上傳去識別資料 |

### DLP 去識別化網閘：讓外部 AI「看不到」機密

這是回應您「最怕員工把客戶名單、財報餵給外部 AI」的核心配置。任何資料在送進外部 LLM／SaaS 之前，必經 **DLP 去識別化網閘**：

- **機密遮罩：** 客戶名稱、財務數字、製程關鍵參數，進外部 AI 前自動代號化（「客戶 A」「製程參數類別 3」），外部 AI 全程看不到真機密。
- **白名單出口：** 預設禁止機密外傳，僅 DLP 放行的去識別資料可出廠。
- **稽核留痕：** 每筆遮罩/還原動作留審計軌跡，供 ISO 42001 治理與外銷客戶供應商稽核舉證。

> **CEO 語言：** 「不是相信外部 AI 廠商不外洩，是根本不讓機密進到外部 AI。沒進去的東西，洩不了。」

## 安控一：Voice AI 的「複誦確認回路」與工法字典治理

回應您「解放滿手油雙手」的核心痛點，語音模組的安全核心是「送出前必有一次人眼把關」：

- **複誦確認（Read-Back Confirm）：** 師傅語音輸入 → 螢幕大字複誦辨識結果 → 看一眼按大鍵確認 → 才寫入系統。報工、報修、規格調整等關鍵指令一律走此回路。
- **領域工法字典：** 建立貴廠專屬的「製程工法語音字典」（機台、工序、規格、料號術語），大幅提升現場噪音與口音下的辨識率；隨新機種持續擴充，版本管理留痕。
- **邊緣離線可用：** 核心辨識部署在廠內邊緣節點，**外網斷線時語音仍可運作**，不讓網路品質成為產線單點故障。

## 安控二：人機協作雙簽（HITL）—— 讓 AI 當加速器，人當守門人

涉及對外、對客、對核心系統的 AI 動作，絕不全自動。分級 HITL：

| 動作類型 | AI 角色 | 人的角色（HITL） |
| :--- | :--- | :--- |
| **唯讀查詢（庫存、工單、客戶歷史）** | AI 直接查詢呈現 | 系統留痕，免人工 |
| **建議性產出（報價草稿、排程建議）** | AI 生成建議草稿 | 承辦複核確認後採用 |
| **寫回 ERP/核心系統** | AI 僅生成待辦 | **強制人工或規則確認後才同步**，AI 無直接寫入權 |
| **對外發送（客戶報告、外銷文件）** | AI 草擬 | **強制人簽核**後方可對外 |

每一筆 AI 動作、每一次人工簽核——誰、何時、做了什麼——全寫入審計日誌，供 ISO 42001 與外銷稽核調閱。

## 安控三：ERP／CRM 不動核心的「三層輕量串接」

回應您的優化建議一，落成可施工的非侵入式架構：

1. **唯讀 API 層：** AI 以唯讀權限讀 ERP/CRM，不寫、不改核心。CIO 評估風險趨近於零。
2. **中介緩衝層（Middleware）：** AI 任何「寫入」需求先進中介層，經人工/規則確認後才同步回 ERP，杜絕 AI 直接污染主資料。
3. **輕量 Agent 補洞層：** ERP 做不到的彈性語音查詢，由 Agentic RAG 補上，不砍掉重練既有系統。

> **退場保護：** 串接採標準 API 與中介層設計，未來更換 AI 供應商時，ERP 核心完全不受影響——**不被任何 AI 廠商綁架。**

## 安控四：成本熔斷與 LLMOps 持續維運

回應您對 TCO 失控的擔憂：

- **單月財務告警閾值：** 每個場景設定單月 API 用量上限與告警，超額自動降級模型或暫停非急件，杜絕帳單黑洞。
- **快取常見問答：** 高頻重複查詢走快取，不重複呼叫 API，直接壓低 Token 成本。
- **LLMOps 分工明確：** 資訊部維護邊緣節點與串接、業務部回饋場景、HR 管教材與培訓，責任清楚不互踢。
