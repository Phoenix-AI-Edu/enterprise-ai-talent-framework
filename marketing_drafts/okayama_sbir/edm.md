### Day 4 信件：傳產變革排除案例

主旨：成功案例分享 — 如何化解一線大師傅的抗拒

各位好，

在推動傳產數位轉型的過程中，抵抗往往來自現場的一線員工。以下是來自「南部扣件大廠」的成功案例，展示他們如何有效化解一線大師傅的抗拒，順利推進 AI 數位轉型。

**案例背景**：
該公司面臨一線大師傅對於新技術的抗拒，主要因為擔心新系統會影響他們的工作流程。

**解決方案**：
1. **教育訓練**：針對所有一線員工提供系統操作的培訓，並邀請他們參與使用者測試，讓他們感受到自己的意見被重視。
2. **語音報工系統**：引入語音報工系統，讓員工可以用自然語言報工，避免了傳統手動輸入的繁瑣，降低操作難度。
3. **激勵措施**：設計「AI 標記教練」職位，將經驗豐富的員工轉型為新系統的教學者，並提供獎金激勵，讓他們感受到轉型的好處。

**結果**：
在實施這些措施後，該公司成功降低了大師傅的抗拒率，並提升了工作效率與數據準確性。

### Day 7 信件：TCO 成本精算避坑指南

主旨：如何精算 RAG 採購成本，避開雲端 SaaS 計費黑洞

各位好，

在推動數位轉型的過程中，了解 TCO（總擁有成本）特別重要，尤其是在處理 RAG 採購時。以下是幾項關鍵要點，幫助您精算成本，避免不必要的開支。

1. **設定預算上限**：
   - 在使用雲端服務時，明確設定每月的預算上限，防止不必要的費用增加。
   - 建議使用預算監控工具，當支出接近預算上限時自動警報，避免超支。

2. **優化 API 請求**：
   - 確保每次 API 請求都有明確的需求，避免不必要的重複請求。
   - 設置 Token 限制，防止因單次請求導致的高額支出。

3. **定期檢討與調整**：
   - 建立定期檢討機制，檢查 API 使用狀況，及時調整使用策略，降低成本。

透過這些策略，您可以有效控制雲端服務的費用，避免進入計費黑洞。如果您有更多疑問或需要進一步的協助，請隨時與我們聯繫。
