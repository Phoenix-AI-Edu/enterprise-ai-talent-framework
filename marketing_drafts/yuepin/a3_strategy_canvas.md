## 🔍 各版塊深度戰略規劃 (Detailed Strategy Breakdown)

### 一、 核心業務痛點與商用價值 (Core Painpoints & Business Value)
1. **隱形 AI 現場排障與查詢（門店收銀/吧台）**：
   - **痛點現況**：外場計時人員流動率極高（平均三個月換一輪），新進計時店員常記不住繁雜的促銷合併規則、退換貨條款與食材履歷，導致收銀異常排隊、引發消費者投訴。
   - **商業價值**：縮短新人完訓上崗時間 3 天，尖峰客流收銀效率提升 20%，免去紙本翻查成本。
2. **智能客服情緒熔斷與交班（客服部）**：
   - **痛點現況**：線上訂位、外帶諮詢大量引入 AI 客服，若發生幻覺承諾不合規退款將引發資金漏洞；且當顧客情緒升溫時，AI 無法安撫，易直接攤在消費者面前，一秒轉為 Google 地圖一星公關負評。
   - **商業價值**：規避 Google 一星公關災難與退款漏洞，AI 過濾 75% 常規提問，減輕真人客服壓力。
3. **智慧人流預測與排班 ML（門店營運/人資）**：
   - **痛點現況**：離尖峰來客量錯配，常憑店長經驗排班，導致離峰人手過剩、尖峰供不應求；且容易違反勞基法工時上限、休假間隔限制，店長每週排班耗時 3 小時以上。
   - **商業價值**：降低門店無效人力成本 5-10%，100% 阻斷勞基法開罰風險。
4. **行銷 GenAI 四道合規防線（行銷部）**：
   - **痛點現況**：行銷與設計團隊大量使用 GenAI 產出文案與影音，但純 AI 獨立生成物不受著作權保護，且面臨風格抄襲、擬真肖像侵權、SaaS 工具授權不清等智慧財產地雷。
   - **商業價值**：保護企業核心品牌資產，規避 IP 與著作權官司風險。
5. **SaaS 供應商治理與安控（總部 IT/財務）**：
   - **痛點現況**：服務業 99% 為採購外部 SaaS AI，面臨供應商 SLA 不穩定（尖峰斷網）、Token 計費黑洞，及供應商倒閉時數據資產轉移等隱形地雷。
   - **商業價值**：保障系統營運不中斷，避免產生高額未知帳單，保障數據所有權不被綁架。

### 二、 技術架構與 Buy vs. Build 決策 (Architecture & Buy vs. Build)
1. **邊緣運算與雲端混網架構 (Hybrid Edge-Cloud)**：
   - **平板邊緣端**：吧台與收銀台平板內建 ASR (語音辨識) 快取，確保外網斷線時，依然能以 Kiosk 靜態模式查詢基本排障指引。
   - **總部私有雲**：排班優化模型與促銷規則 RAG 知識庫部署在總部本地伺服器，保障敏感營收人流與配方數據 100% 留廠。
   - **雲端 API 網道**：線上客服對話與翻譯採用公有雲大模型 API，但必須經過 DLP 網閘進行個資過濾與字數/金額限制。
2. **Buy vs. Build 決策定義**：
   - **Build (自主研發/核心護城河)**：排障語音 ASR（與悅品專有餐飲語彙深度綁定）與排班預測 ML 模型（悅品專有來客特徵）。
   - **Buy/Rent (軟體採購/快速集成)**：行銷 GenAI 工具（使用主流白名單 SaaS）與線上客服 API（引入 GPT/Gemini 進行對話，但自建安全閘門）。

### 三、 資料資產與安全防護防線 (Data Assets & Security Guardrails)
1. **專有資料資產沉澱**：
   - 收集門店一線雙國台語、餐飲配方、促銷術語，建立「悅品專有服務語音字典與知識庫」。
   - 累積歷史 POS 交易數據、人流波形特徵，沉澱為具有競爭優勢的餐飲排班預測時序模型。
2. **三道安全防線**：
   - **DLP 個資網閘**：進 LLM 前自動將姓名、電話、會員卡號遮罩成代碼，AI 產出後再還原。
   - **退款權限系統硬上限**：在系統層鎖死 AI 退款權限上限，AI 僅能生成建議，不開放核准權限。
   - **情緒熔斷機制**：一秒偵測敏感詞與情緒升溫信號，AI 立即熔斷，自動生成對話摘要並推送給真人客服無縫接班。

### 四、 試點計畫與擴展時程 (Pilot Plan & Scaling Timeline)
1. **第一階段：Quick-Win 試點（M1-M3）**：
   - **試點範圍**：悅品中餐廳、悅品點心 3 家旗艦店（計時人員共 30 人）實施語音排障平板；線上客服組實施情緒熔斷客訴分流。
   - **成功指標**：語音辨識率 ≥92%，情緒熔斷成功率 100%，計時人員完訓時間縮短 3 天。
2. **第二階段：智慧排班與行銷合規落地（M4-M6）**：
   - **試點範圍**：指定 10 家門店實施 ML 排班；行銷部導入四道合規閘門工作流。
   - **成功指標**：排班配置誤差率降低 30%，店長排班工時降至數分鐘，發布行銷素材 100% 合規。
3. **第三階段：全廠擴展與政府補助（M4-M6 並行）**：
   - 備齊 ≥10 店的試點佐證照片與 3 個月效益數據，送件申請中企署 SIIR（服務業創新研發計畫）連鎖體系最高 500 萬補助，並對接產創條例租稅抵減。

### 五、 變革管理與人機協作設計 (Change Management & HITL Design)
1. **有人在環 (Human-in-the-Loop, HITL) 設計**：
   - **現場排障 (Copilot)**：AI 檢索答案，計時店員進行二次確認才答覆顧客。
   - **線上客服 (Validator)**：AI 回覆一般問題，客訴與退款一律由真人客服熔斷拍板。
   - **門店排班 (Validator)**：排班優化模型提供合規建議班表，店長保留 100% 最終手動修改與核准拍板權。
2. **組織變革心理化解方案**：
   - **安全免責與微課培訓**：拍製 3 分鐘情境微影音完訓即上崗，並設定系統免責期，消除時薪人員對工具的焦慮與排斥。
   - **人機無縫交班**：熔斷時 AI 自動生成摘要推送給專員，店長排班工時節省納入考核，讓員工體驗到 AI 是「省力幫手」而非工作威脅。
   - **利益同頻**：總經理發表「不裁員安心承諾書」，並舉辦 Prompt 效率大賽給予實質獎金。

### 六、 營運治理與持續維運 (Ops & Governance)
1. **LLMOps 持續維運分工**：
   - **IT 部門**：主責 API 流量、網閘安全、白名單權限及 Gateway 財務斷路器維護。
   - **營運與行銷部門**：每週收集未辨識語音、新促銷漏洞及行銷相似度報告，進行模型字典與 RAG 知識庫微調。
2. **財務安控與熔斷防護**：
   - 全廠 API 流量費財務告警總閾值設為 **NT$ 50,000 / 月**。
   - 當單月 API 費用達到預算 90% 時，Gateway 自動熔斷並將非敏感一般詢問導向靜態 FAQ，保留 API 額度給高敏顧客，防堵帳單黑洞。
