## Slide 03：CFO 的硬核財務公式：TCO 拆解
*   **投影片標題**：CFO 的 AI 總擁有成本 (TCO) 全景拆解與財務精算
*   **視覺設計建議**：
    *   **版面配置**：一張乾淨、現代的 Excel 精算表圖形。
    *   **核心財務公式**：
        $$\text{Total TCO} = \sum_{t=1}^{N} \frac{\text{API}_{t} + \text{Infra}_{t} + \text{Eng}_{t} + \text{Audit}_{t}}{(1 + r)^t}$$
    *   **各變數定義卡**：
        *   $\text{API}_{t}$ ➔ Token 消耗費（分詞、上下文通膨）。
        *   $\text{Infra}_{t}$ ➔ 伺服器/地端 GPU/向量庫維護成本。
        *   $\text{Eng}_{t}$ ➔ 持續特徵工程與模型重訓/漂移修正人月。
        *   $\text{Audit}_{t}$ ➔ 安全漏洞與合規賠償準備金。
    *   **警示文字**：`「只算 API Token 月費，就像買跑車只算加油錢，卻忽略了天價的折舊與維修保養」`。

```text
請大家看投影幕上，這是我們鳳凰 AI 顧問團隊為 C-Suite 決策者制定的「AI 總擁有成本 (TCO) 加權精算公式」。
```

## Slide 04：失控病灶一：Token 分詞地雷
*   **投影片標題**：Token 3 倍通膨地雷：繁中分詞機制與 API 成本精算

        *   *優化後繁中*：`"創新人工智慧解決方案。"` ➔ 鳳凰專屬 Tokenizer 優化後切成 4 個 Token ➔ 成本降至 **130%**。
    *   **防坑條款**：`「在採購任何 AI 系統前，要求廠商出具繁中 Token 壓縮比測試報告」`。

```text
陳策略長（Phoenix）：
我現在用技術本質為大家解構第一個預算殺手：繁中 Token 3 倍通膨。

英文 `"Innovative AI solutions."` 對於模型來說，結構非常簡單，只需 3 個 Token。

陳策略長（Phoenix）：
為了解決這個硬傷，我們鳳凰 AI 在為企業進行特徵工程與模型對接時，會強制加裝「繁中分詞優化網閘」。
我們在數據送入大腦前，先以本地端轻量化演算法進行詞意預打包，把「人工智慧」這四個字合併為單一特徵 Token 發送。
光是這一項底層的特徵工程優化，就能幫公司直接砍掉 60% 的繁中提問 Token 費用。這就是技術直覺帶來的財務利潤。
```

        *   下午 5 點：問答 20 ➔ 瘋狂膨脹至 40,000 Token！
    *   **防禦機制卡片**：亮綠色高亮「滑動窗口 (Sliding Window) 裁剪」與「地端語意摘要過濾器 (Summarizer Agent)」。

### 🎤 雙顧問現場輪講逐字手稿

```text
陳策略長（Phoenix）：
第二個失控病灶，是我們在 M05 談到 RAG 時也常遇到的「上下文無限累積」。
大模型是沒有長期記憶的，它每次之所以能回答您：「承上題，那第二點是什麼？」，是因為系統把您先前的所有提問和 AI 的回答，全部作為「脈絡 (Context)」重新餵給它。

孟顧問（Meng）：
我們來算一筆帳。同仁早上 9 點開一個視窗問了第一句 SOP 規則，消耗 500 個 Token，大約新台幣 0.1 元。
到了下午 5 點，他在同一個視窗問第 20 個問題時，歷史對話已經累積到了 4 萬個 Token。
這時候，他輸入一句簡單的「好的，請幫我存檔」，後端大模型為了看懂「好的」這兩個字，必須把前面 4 萬個 Token 全部重新閱讀並運算一次。

陳策略長（Phoenix）：
這句「好的」，單次運算成本瞬間暴增到新台幣 8 元！
這就是為什麼您的 API Gateway 流量儀表板上，會出現一堆毫無商業價值的「天價垃圾提問」。
我們的解決方案是部署「滑動窗口過濾器」。
在 API 網閘端限制每次對話最多只攜帶前 3 輪的歷史紀錄，並由一個地端極其便宜的輕量化 Agent，在後台自動將 3 輪之前的對話提煉成 100 字的精簡摘要。
這能讓您的上下文 Token 消耗量永遠維持在低位平穩，徹底告別滾雪球帳單。
```

```text
我們必須在 API Gateway 端，加裝「意圖智慧分類路由 (Router)」，自動把小鳥分給 Flash 模型，把真正的跨文件大專案才路由給頂級大腦。這能為您瞬間砍掉大筆模型開銷。
```
