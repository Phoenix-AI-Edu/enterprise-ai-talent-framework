# 🎨 「4+1 企業 AI 戰略畫布」Notion 互動模板規格指南

## 📋 主表一：企業 AI 場景盤點資料庫 (Scenario Discovery DB)
*   **定位**：盤點業務痛點，透過公式自動篩選出高價值、低阻力、高數據就緒度的 **Quick-Win (快速獲勝) 試點專案**。

### 1. 欄位設計與屬性 (Properties)：
| 欄位名稱 | 屬性類型 (Type) | 設定值 / 選項範圍 (Configuration) | 說明 |
| :--- | :--- | :--- | :--- |
| **業務痛點 (Title)** | 標題 (Title) | *如：售後技術客服查手冊慢* | 具體描述痛點，非模型技術。 |
| **主責部門** | 單選 (Select) | 客服 / 製造 / 財務 / 行銷 / HR / 研發 / 行政 | 業務痛點對應的部門。 |
| **核心 AI 任務** | 單選 (Select) | Q&A (問答) / Classification (分類) / Summary (摘要) / Prediction (預測) / Generation (生成) | 技術任務分類。 |
| **商業價值評估 (V)** | 單選 (Select) | 1-低價值 (省時微小) <br> 3-中價值 (省時顯著/輔助性) <br> 5-高價值 (直接提升 ROI/節省關鍵工時) | 評估導入後的財務與效率收益。 |
| **數據就緒度評估 (D)** | 單選 (Select) | 1-無數據 / 2-紙本或極零散 / 3-數位化未整理 / 4-結構化 PDF 或資料庫 / 5-高 API/向量庫就緒 | 資料的可存取性與完整度。 |
| **一線使用阻力評估 (R)**| 單選 (Select) | L - 低阻力 (員工極渴望改善) <br> M - 中阻力 (需要簡單調整習慣) <br> H - 高阻力 (員工擔心失業/流程極複雜) | 一線人員排斥或抵制的風險。 |
| **ROI 綜合評估指數** | 公式 (Formula) | *見下方 Formula 2.0 設計* | 自動計算的分數 (總分 1.0 ~ 5.0)。 |
| **導入優先級決策** | 公式 (Formula) | *見下方 Formula 2.0 設計* | 自動判定是否為優先 Quick-Win 項目。 |

### 2. 核心公式設計 (Notion Formula 2.0)：

#### 🟢 ROI 綜合評估指數：
將商業價值（權重 50%）、數據就緒度（權重 30%）與使用阻力（權重 20%，H 扣分最多）進行加權計算：
```javascript
/* Notion Formula 2.0 語法 */
let(V_val, toNumber(left(prop("商業價值評估 (V)"), 1)),
let(D_val, toNumber(left(prop("數據就緒度評估 (D)"), 1)),
let(R_score, if(left(prop("一線使用阻力評估 (R)"), 1) == "H", 1.0, if(left(prop("一線使用阻力評估 (R)"), 1) == "M", 3.0, 5.0)),
round(((V_val * 0.5) + (D_val * 0.3) + (R_score * 0.2)) * 10) / 10
)))
```

#### 🔵 導入優先級決策：
依據 ROI 指數自動打星，便於高階主管決策：
```javascript
/* Notion Formula 2.0 語法 */
if(prop("ROI 綜合評估指數") >= 4.0, "⭐⭐⭐ 優先導入 (Quick-Win)", 
if(prop("ROI 綜合評估指數") >= 2.8, "⭐⭐ 中期規劃 (需補強數據/化解阻力)", 
"⭐ 暫緩/儲備項目"))
```

## 📋 主表二：資料基礎與方案架構資料庫 (Data & Security DB)
*   **定位**：對齊 ISO/IEC 42001 安全標準，規劃專案的技術路徑與主動安全防禦防線。

### 1. 欄位設計與屬性 (Properties)：
| 欄位名稱 | 屬性類型 | 設定值 / 選項範圍 | 說明 |
| :--- | :--- | :--- | :--- |
| **對應場景專案** | 關聯 (Relation) | 關聯至 `📋 主表一：場景盤點資料庫` | 建立專案與技術架構的一對一關係。 |
| **技術路徑選擇** | 單選 (Select) | **SaaS 採購 (Buy)**：如 Microsoft 365 <br> **API 客製 (Build)**：以 OpenClaw 串接 Gemini/Claude <br> **私有化部署 (Build)**：內網部署 Llama-3-70B | 對應 Buy vs. Build 決策。 |
| **核心模型配置** | 多選 (Multi-select) | Gemini 1.5 Pro / Claude 3.5 Sonnet / Llama-3.1 / GPT-4o | 專案採用的基礎或開源模型。 |
| **敏感資料防線** | 多選 (Multi-select) | 🛡️ 個資去識別化遮罩 <br> 🛡️ 本地端 DLP (資料外洩防護) <br> 🛡️ 傳輸邊界加密 (SSL/TLS) <br> 🛡️ 靜態資料庫加密 (AES-256) | 保障企業機密不外洩。 |
| **主動安全防禦** | 多選 (Multi-select) | 🧱 System Instruction 強制隔離 <br> 🧱 Guardrails 防護欄規則 <br> 🧱 Prompt Injection 主動偵測機制 | 阻絕惡意提示詞注入。 |
| **合規審計稽核** | 多選 (Multi-select) | 📑 使用者 ID 綁定記錄 <br> 📑 Token 呼叫流量限制 <br> 📑 全量對話歷史審計存檔 | 配置完整的 Audit Trails（審計軌跡）。 |

## 📋 主表三：試點驗證與阻力評估資料庫 (Pilot & Change DB)
*   **定位**：強制導入「變革管理 (Change Management)」，預先評估一線人員抵制，並規劃應對行動。

### 1. 欄位設計與屬性 (Properties)：
| 欄位名稱 | 屬性類型 | 設定值 / 選項範圍 | 說明 |
| :--- | :--- | :--- | :--- |
| **對應場景專案** | 關聯 (Relation) | 關聯至 `📋 主表一：場景盤點資料庫` | 綁定對應的業務專案。 |
| **試點部門與範圍** | 文字 (Text) | *如：高雄二廠售後組 (10 人)* | 明確試點的物理邊界與人數。 |
| **試點週期** | 單選 (Select) | 14 天快速迭代 / 30 天深度驗證 / 60 天全面觀察 | 設計試點的時程限制。 |
| **Quick-Win 量化 KPI** | 文字 (Text) | *如：客戶問題即時解決率從 60% 提升至 90%* | 設定可量化且極易達成的目標。 |
| **一線員工主要阻力** | 多選 (Multi-select) | ⚠️ 焦慮恐懼被 AI 取代/失業 <br> ⚠️ 抱怨系統繁瑣增加工作量 <br> ⚠️ 抱怨模型幻覺/對系統不信任 <br> ⚠️ 傳統操作習慣難以改變 | 識別阻力來源，作為變革起點。 |
| **變革管理化解方案**| 多選 (Multi-select) | 💡 導入語音輸入/UI 一鍵化極簡設計 <br> 💡 舉辦 Prompt 效率競賽提供獎金獎勵 <br> 💡 設計 HITL 安全隔離免除員工出錯責任 <br> 💡 管理層宣示承諾「不因 AI 裁員」 | 具體的組織心理變革配套措施。 |

## 📋 主表四：營運、治理與組織變革資料庫 (Ops & Governance DB)
*   **定位**：界定人機協作決策權責，建立財務防禦熔斷閾值，規劃持續維運體系。

### 1. 欄位設計與屬性 (Properties)：
| 欄位名稱 | 屬性類型 | 設定值 / 選項範圍 | 說明 |
| :--- | :--- | :--- | :--- |
| **對應場景專案** | 關聯 (Relation) | 關聯至 `📋 主表一：場景盤點資料庫` | 關聯至場景。 |
| **人機協作角色設計**| 單選 (Select) | 🟢 **Autonomous** (AI 100% 執行/低風險) <br> 🟡 **Copilot** (AI 出草稿，人類覆核/中風險) <br> 🔴 **Validator** (人類 100% 裁決，AI 提供建議/高風險) | 精準配置決策權責 (Human-in-the-loop)。 |
| **單月 API 財務告警閾值**| 數字 (Number) | 數值格式：USD (美元) / TWD (台幣) | 超過此數值系統需自動告警或熔斷。 |
| **持續維運分工 (LLMOps)**| 單選 (Select) | IT 部門主責 (維護向量庫與主機) <br> 業務部門主責 (資料勘誤與提示詞調優) <br> 外部服務商主責 (全量維託管) | 防止專案上線後因無人維護而死。 |
| **技能升級與培訓** | 單選 (Select) | 1 天極簡導入訓 / 3 天 Prompt 實戰訓 / 5 天跨部門菁英開發訓 | 一線人員技能重塑（Upskilling）規劃。 |
| **財務熔斷保護** | 核取方塊 (Checkbox)| [ ] Checked / [ ] Unchecked | 若 Checked，代表已將告警腳本寫入 API Gateway。 |
