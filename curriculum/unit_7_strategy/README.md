## 📝 核心課題模組快覽

### 1. 為什麼 80% 的 AI 專案最終死於 POC？
* **四大陷阱**：深剖玩具陷阱 (Toy Trap)、資料與系統孤島 (System Silo)、組織變革壁壘 (Change Resistance) 與營運成本失控 (TCO Explosion) 的痛點本質。

### 2. 變革管理 (Change Management) 與人機協作
* **阻力應對**：化解「恐懼被取代」與「增加工作負擔」的心理防線。
* **人機協作比例 (HITL)**：劃分 Autonomous (全自動)、Copilot (副駕駛) 與 Validator (最終裁決者) 的決策權。

### 3. Buy vs. Build (自研與採購) 決策矩陣
* **決策樹與權衡**：精準權衡上線時間、初期建置成本 (CapEx)、長期營運成本 (OpEx)、客製化彈性以及安全合規要求。

### 4. 「鳳凰AI」整合規劃畫布包 (4張主表 + 1張總表)
* **場景盤點畫布**、**資料與架構畫布**、**試點與阻力畫布**、**營運治理畫布**與**一頁式總體戰略畫布**的實兵演練。
