# 👁️ 單元五：鑑別式 AI 的原理與應用控制層

本單元將鑑別式 AI 重新定位為 **「AI Agent 的中央路由閘門與安全控制層」**。透過條件機率學習與可解釋性分析技術，在多代理人（Multiagent）系統的前端與核心決策端建構堅實的防黑天鵝防線。

---

## 🎯 學習目標
* **釐清數學本質**：深入理解鑑別式 AI 的條件機率 $P(Y \mid X)$ 與生成式 AI 聯合機率 $P(X, Y)$ 的本質區別。
* **建構安全控制閘門**：學會如何以鑑別式 AI 作為大模型前置的「意圖分類器（Intent Classifier）」與「異常偵測器」，實現安全的分流與 Human-in-the-loop (人工覆核)。
* **掌握模型可解釋性**：精通 SHAP (Shapley 值) 與 LIME 的運作原理，學會在金融信貸與醫療診斷中輸出符合合規要求的「決策歸因報告」。

## 📝 核心課題模組快覽

### 1. 條件機率與數學邊界
* **對比模型**：鑑別式學習條件機率 $P(Y \mid X)$，尋找優勢分類邊界；生成式學習聯合機率 $P(X, Y)$，重構整體數據分佈。

### 2. 意圖路由與分佈外檢測 (OOD)
* **路由網關**：前置分類器識別高風險意圖（退款、客訴），繞過 LLM 自動對答，直接分流至 Human-in-the-loop 真人覆核。
* **OOD 異常判定**：利用特徵距離檢測異常與惡意注入，降低大模型無謂的算力開銷與合規威脅。

### 3. 可解釋性 (Explainability) 與法規合規性
* **SHAP 技術**：博弈論 Shapley Value 與特徵歸因分析公式，為黑盒子算法提供完美的全局與局部特徵貢獻解釋。
* **LIME 技術**：利用局部線性擬合，解決局部決策的可解釋性瓶頸，滿足 GDPR 獲得解釋權的嚴格規範。
