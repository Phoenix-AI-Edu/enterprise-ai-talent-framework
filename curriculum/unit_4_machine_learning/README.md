## Slide 06：多模態特徵工程一：滑動窗口統計與差分特徵
*   **投影片標題**：時間序列特徵工程：滑動窗口統計與差分特徵實務
*   **視覺設計建議**：
    *   **版面設計**：左側展示原始的時序波動線圖；右側展示經過特徵工程處理後的「特徵矩陣表格」。
    *   **動畫指引**：一個半透明的金色「滑動窗口」在時序波形上緩慢移動，同步在右側表格中填入滑動均值 $\mu_t$ 與滑動標準差 $\sigma_t$。
    *   **公式呈現**：清晰美觀的 LaTeX 數學公式置於卡片中央。

### 🎤 雙顧問現場輪講逐字手稿

```text
陳策略長（Phoenix）：
在機器學習界有一句至理名言：「數據與特徵工程決定了模型精度的上限，而演算法只是在逼近這個上限而已。」
很多工程師在做時序預測時，直接把感測器的原始數據餵給 LightGBM 或 XGBoost，結果精度慘不忍睹。因為樹模型本質上是分裂算法，它無法直接理解數值在時間維度上的「趨勢」與「波動」。
因此，我們必須手工建構特徵。第一個經典特徵是「滑動窗口統計 (Rolling Window Statistics)」。

我們在原始序列上開闢一個長度為 $W$ 的滑動窗口，實時計算窗口內的滑動均值 $\mu_t$ 與滑動標準差 $\sigma_t$：

$$\mu_t = \frac{1}{W} \sum_{i=0}^{W-1} x_{t-i}$$
$$\sigma_t = \sqrt{\frac{1}{W} \sum_{i=0}^{W-1} (x_{t-i} - \mu_t)^2}$$

這兩個特徵能幫模型捕捉到：這台機器的震動訊號是正在緩慢上升（均值增加），還是波動正在變得劇烈（標準差暴增）？

孟顧問（Meng）：
這在商業上同樣有巨大的應用。例如預測超市蔬菜銷量時，過去 7 天的滑動銷量均值代表了基礎需求，而滑動標準差則代表了市場波動。

陳策略長（Phoenix）：
另一個特徵是「差分特徵 (Lag/Difference Features)」：

$$\Delta x_t = x_t - x_{t-k}$$

它計算的是當前數值與前 $k$ 個時間步的差值。這能有效消除數據的非平穩性，捕捉到如高能耗設備瞬間啟動時的電壓驟降，或者突然爆發的促銷需求。
```

    *   **右側圖表**：頻域光譜圖。透過金色 FFT 變換曲線，在 120Hz 處呈現一根極高聳、耀眼的金色能量特徵峰，背景發出藍色科技光暈。

### 🎤 雙顧問現場輪講逐字手稿

```text
陳策略長（Phoenix）：
如果剛才的滑動窗口是時序的「形」，那麼我們現在要談的就是時序的「魂」——頻域特徵工程。
在智慧製造的「預測性維護 (PdM)」中，我們最頭痛的是設備的機械磨損。以半導體晶圓廠蝕刻機台的真空幫浦（Dry Pump）為例，當轉軸軸承（Bearing）開始出現微小磨損時，其震動幅度非常小，在時域訊號上看起來與平時完全一致，傳統模型根本無法識別。
但這時候，如果我們使用「快速傅立葉變換 (FFT)」將時域訊號轉換至頻域：

$$X(f) = \int_{-\infty}^{\infty} x(t) e^{-i 2\pi f t} dt$$

奇蹟就發生了。

孟顧問（Meng）：
各位可以看到投影片右側的頻域光譜。
原本在左邊雜亂無章的時域波形，在右邊經過 FFT 變換後，會在特定的特徵故障頻率（例如 120Hz 及其高次諧波）上，聳立起一根巨大的金色特徵峰！這根特徵峰的能量比正常運轉時暴增了數十倍。
```
