# 📌 單元四：機器學習技術理論與實務案例

本單元修正舊版只強調監督/非監督式學習算法公式，或盲目進行 70/15/15 資料集分割實驗的 IT 思維，改為聚焦於企業導入機器學習時的 **ROI 進化路線**。

---

## 🎯 學習目標
* 掌握企業機器學習專案的正確建置與演進順序。
* 理解特徵工程 (Feature Engineering) 在實際工業場景中的應用。
* 理解多模態數據融合（數值 + 影像 + 文本）在 2026 年的核心挑戰。

---

## 📖 核心知識模組

### 1. 企業機器學習的正確建置順序 (ROI 演進路線)
教材必須強制灌輸學員，不要一開始就盲目投資自研或從零訓練（Pretrain）大模型，應遵循以下由淺入深的 ROI 進化路線：
```text
使用現成模型 API 
   └── 建立 RAG 企業知識庫 
         └── 導入 AI Agent 流程自動化 
               └── 視需求進行微調 (Fine-tuning) 
                     └── 最後在極端資安/傳產專用場景下才考慮「私有部署自研模型」
```

### 2. 特徵工程與多模態數據融合
現代機器學習案例（如：智慧能源管理、工廠預測性維護）已不僅僅看單一數值。系統必須同時處理：
* **結構化表格資料**（例如：歷史電壓、溫度、震動頻率）。
* **現場監視器影像**（例如：設備外觀紅外線熱成像）。
* **老師傅的維修手記文本**（例如：非結構化的故障描述紀錄）。

如何進行多模態資料的特徵提取與融合，是企業資料科學家的核心課題。

---

## 📝 課後練習與實務思考
1. **策略抉擇**：若一家傳統製造業希望建立「廠房電力負載預測系統」，請對照「ROI 進化路線」，為其規劃第一階段與第二階段的具體評估指標，並說明為什麼不應該一開始就自行開發模型。
2. **多模態特徵思考**：試舉出一個「智慧零售」或「智慧醫療」的應用場景，列出其包含的結構化、影像、以及文本資料，並討論如何將其融合成單一預測模型的輸入。
