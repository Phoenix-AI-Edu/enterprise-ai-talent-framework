### Slide 03｜2026 語音技術革命：傳統串接 vs. 端到端 Realtime API
* **【版面設計】**：清晰的對比表格。左側展示傳統語音系統的「串接式卡頓架構」，右側展示 2026 全新「端到端 Realtime API 架構（發金色光芒）」。
* **【核心標題】**：**技術代價 ｜ 跨越 3 秒的致命遲滯**
* **【核心內容】**：
  * **傳統串接架構 (STT ➔ LLM ➔ TTS)**：
    *   *體驗* ➡️ 延遲高達 **3 - 8 秒**。說完話必須等待，音色生硬像機器人，**無法打斷**。
    *   *結果* ➡️ 現場師傅完全失去耐心，誤判定為系統死機。
  * **2026 端到端 Realtime API (Gemini Live / OpenAI Realtime)**：
    *   *體驗* ➡️ 延遲僅 **0.5 - 1.2 秒（與真人對話無異）**。
    *   *特點* ➡️ 音色充滿抑揚頓挫與情緒，**隨時支持說話打斷 (Interruption)**。
* **【主講人】**：陳文家策略長
* **【時間管控】**：180 秒 (累計 07 分鐘)
* **【講師逐字稿】**：
  「各位先進好，我是陳文家。接下來我從軟體工程的角度，為大家解析為什麼 2026 年是 Voice AI 落地傳產的『真元年』。

  在過往，語音系統是拼湊起來的：先把語音轉成文字 (STT)，丟給大模型 (LLM)，再把模型產出的文字轉成語音 (TTS)。這一來一回，產生了高達 5 秒以上的致命延遲。更糟的是，大模型在喋喋不休地說規格時，技師想打斷它、問下一個步驟，系統根本聽不到，必須等它整段唸完。這種體驗在分秒必爭的工廠裡是不可接受的。

  2026 全新上線的端到端語音模型（如 Gemini Live API），在底層將音訊直接映射為高維度向量。

  **延遲直接被砍到了 0.5 秒，且支援隨時語音中斷。** 當 AI 在唸手冊時，您直接說『停！告訴我下一個螺帽怎麼鎖』，系統在毫秒內立刻中斷並回覆。這才是現場技師要的『智慧助手』。」

### Slide 04｜第一道關卡：85dB+ 高噪環境下的陣列降噪麥克風與邊緣端 VAD 語音檢測
* **【版面設計】**：深色背景，左側為一個轟鳴的沖床 3D 模型（標記 85dB - 90dB 聲波紅圈），中間為「陣列降噪過濾器」，右側為乾淨的「人類語音活動檢測 (VAD) 波形圖」。
* **【核心標題】**：**第一道防線 ｜ 噪聲屏障與 VAD 機制**
* **【核心內容】**：
  * **廠區致命噪聲** ➡️ 沖床、鍛造與精密磨床的機械噪音常態大於 **85分貝 (dB)**，導致普通語音軟體完全癱瘓。
  * **解決方案**：
    1.  **硬體防護** ➡️ 部署主動降噪（ANC）骨傳導耳機或陣列式降噪定向麥克風。
    2.  **軟體過濾** ➡️ 部署地端邊緣 VAD (Voice Activity Detection) 機制，利用特徵工程在邊緣端直接砍掉高頻碰撞與背景底噪，只傳輸人類純語音。
* **【主講人】**：陳文家策略長
* **【時間管控】**：180 秒 (累計 10 分鐘)
* **【講師逐字稿】**：
  「當我們決定要用語音解決交班問題時，我們碰到的第一個硬核技術關卡，就是**『工廠噪音』**。

  南部的螺絲打頭機、鑄造廠行車，現場背景噪音常態性超過 85 分貝。如果直接用普通手機麥克風，外部 API 會把機械的轟鳴聲全部錄進去，大模型只會收到一堆亂碼。

  鳳凰 AI 的解決方案是『軟硬雙控』。在硬體端，我們不強求工廠全面降噪，而是讓師傅配戴我們特製的骨傳導主動降噪耳麥；

  在軟體端，我們在地端網關部署了**輕量級 VAD 語音活動檢測**。它能在語音封包發送前，在邊緣端自動將機械的規律高頻碰撞聲物理過濾，只在判定『有人類開口說話』時才啟動 API 傳輸。這不僅能將辨識精準度拉到 93% 以上，更能幫您省下 80% 的無效流量費用。」

### Slide 05｜第二道關卡：方言與口音辨識（國台語混雜口音與南部產業術語實例）

* **【核心標題】**：**第二道防線 ｜ 國台語與產業黑話大腦**
* **【核心內容】**：
  * **南部傳產黑話** ➡️ 一線師傅說話習慣國台語混雜，且夾帶特有的日式口音或台式英文英文術語。
  * **典型口語實例**：
    > *「這台 controller 目前有 lens 誤差，愛去 check 一下」*
  * **鳳凰 AI 解決方案**：
    *   在推論前端掛載 [自研之語音編碼與術語庫對照機制](file:///g:/%E6%88%91%E7%9A%84%E9%9B%B2%E7%AB%AF%E7%A1%AC%E7%A2%9F/AI_Talent/curriculum/unit_2_industries/curriculum_v2026.md)，將日式日文黑話（如：阿嚕米、露兜）與英文縮寫進行預處理對應。
* **【主講人】**：陳文家策略長
* **【時間管控】**：180 秒 (累計 13 分鐘)
* **【講師逐字稿】**：
  「這是我們中南部工廠最真實的文化。

  現場師傅是不可能對著機器說標準國語的。他們交班時，通常是：*『文家啊，跟你講一下，那台 controller 目前有點 lens 誤差，等一下愛去 check 一下喔。』*

  這裡面包含了三個挑戰：第一，『愛去』是台語；第二，『controller』和『lens』是台式英文；第三，『check』是夾雜術語。如果是普通的美國大模型，辨識出來的文字會是：*『這台控制器有冷死誤差，愛去缺口一下。』* 委員一聽，肯定直接打回票。

  鳳凰 AI 在本地中台的前端，掛載了專為台灣南部製造業打造的**『台式英文與日式黑話對照庫』**。

  不論是『阿嚕米 (鋁)』、『阿達媽 (機頭)』還是『Lens』，我們的編碼器會在送給 LLM 之前，自動將其對應到標準的物料編碼與 technical term 中，這才是真正符合工廠文化的在地化大腦。」

### Slide 06｜第三道關卡：個資保護與台灣個資法語音強制主動告知
* **【版面設計】**：警戒的黃色底色。左側為「台灣個人資料保護法語音規定卡片」，右側為 AI 語音助理啟動時的標準「10秒強制播音宣告」流程圖。
* **【核心標題】**：**第三道防線 ｜ 語音敏感個資合規網閘**
* **【核心內容】**：
  * **語音即個資** ➡️ 依照台灣個人資料保護法，聲音特徵與錄音檔屬於具備唯一識別性的「敏感個資」。
  * **技術強制告知 SOP**：
    *   凡是涉及與外部客戶通話、或一線員工錄音之系統，**對話前 5 秒必須自動播送標準語音告知**：
    > 「【鳳凰 AI 提示】您好，為了提供智慧分析與維運安全，本次通話將進行加密記錄與語音去識別化。若您繼續通話即代表您同意本公司履行個資法第 8 條之告知同意義務。」
  * **地端物理銷毀** ➡️ 錄音檔在轉譯成結構化 JSON 文本後，**於地端安全存儲伺服器內定時 24 小時強制擦除**，防範二次外洩風險。
* **【主講人】**：陳文家策略長
* **【時間管控】**：120 秒 (累計 15 分鐘)
* **【講師逐字稿】**：
  「在我們部署語音系統時，第三個最容易被企業法務忽視的地雷，就是**個資法對『語音』的嚴格監管**。

  聲音具有生物唯一性，在法律上屬於敏感個資。如果您的語音助理在沒有告知客戶的前提下，就開始錄音並轉成文字，一旦發生個資外洩，公司面臨直接起訴。

  我們為企業建置的系統中，強制埋入了**語音合規網閘**。

  第一，在對話前，系統會強制播送 10 秒鐘的個資法第 8 條宣告，由使用者用語音回覆『同意』或繼續對話來建立法律免責憑證。

  第二，我們的語音檔案絕對不會放在雲端伺服器上。一旦 STT 引擎將師傅的口述轉譯成結構化的 JSON 日誌，**地端網閘會在 24 小時內啟動物理擦除，只保留不含聲音生物特徵的去識別化數據。** 這就是最嚴密的法規安全設計。」

* **【核心標題】**：**【實作環節 1】語音智慧轉譯現場挑戰 (3 分鐘)**

  * **現場 Live 展示**：
    1.  AI 在 1.2 秒內完成低延遲轉譯。
    2.  將混亂的口語，自動抽離並結構化為符合 ERP 標準的 **`設備編號`**、**`瑕疵等級`**、**`微調步驟`** 與 **`主責人員`**。
* **【主講人】**：孟淑慧首席顧問 ＆ 陳文家策略長

* **【講師逐字稿】**：
  「現在，是眼見為憑的實作時間了。

  我們邀請現場兩位最懂現場運作的先進，上台拿起我手上的這台對講機，試著用您們廠區最真實、最隨意的口語，模擬一段 30 秒的交班對話。

  您可以隨意夾雜台語、英文，甚至故意製造一些贅詞。

  【學員 A 模擬：*『那台 04號打頭機今天送料夾怪怪，我剛才已經叫志明去調了啦，下班愛去check一下。』*】
