### Slide 06｜技術對焦一：RAG 引用追溯（Citation Tracing）與事實一致性（Faithfulness）網閘
* **【版面設計】**：明亮對比的資料流圖。展示向量庫檢索 Chunk 標註 Doc ID ➔ 填充至 Prompt ➔ 大模型生成帶標籤回覆 ➔ Faithfulness Guardrails 網閘 10毫秒內進行 NLI 推理與一致性驗證。
* **【核心標題】**：**安全防禦 ｜ RAG 事實一致性網閘**
* **【核心內容}】**：
  * **引用追溯 (Citation Tracing)** ➡️ 在 RAG 生成的每句結尾，強制標註來源標籤（如 `[Source: Chunk_99]`）。
  * **事實一致性驗證 (Faithfulness Check)** ➡️ 本地閘道在 **10 毫秒** 內比對 AI 生成文本與原始 Chunk 語意是否一致。
  * **安全熔斷** ➡️ 一旦偵測到模型自我幻覺、胡說八道，網閘立刻阻斷輸出，重啟檢索或降級為真人客服。
* **【主講人】**：陳文家策略長
* **【時間管控】**：180 秒 (累計 15 分鐘)
* **【講師逐字稿】**：
  「這是我們部署在企業內網的**第一道可解釋性防線：事實一致性網閘**。

  大模型很容易產生幻覺，俗稱『一本正經地胡說八道』。

  我們強制的 Citation Tracing 機制，會在 RAG 生成的每個語句結尾，強制塞入它引用的 Doc ID 與區塊編號。

  更重要的是，我們在出口端放了一個 **Faithfulness 驗證網閘**。它會用一個微型的自然語言推理模型，比對大模型產出的句子，跟原始 Chunk 原文的語意是否吻合。

  如果大模型自己編造了數據，網閘會在 10 毫秒內直接物理熔斷，拒絕將這個回答發送給客戶，並退回重啟。**這意味著：您給客戶的答案永遠『有據可查』，徹底封鎖了幻覺帶來的法律曝險。**」

### Slide 07｜技術對焦二：博弈論 SHAP (Shapley 值) 的數學邊際貢獻分解公式
* **【版面設計】**：深色背景。中央以發光效果呈現 SHAP 可加性公式與 Shapley 值積分公式，下方配有特徵邊際貢獻的發光箭頭（紅色代表拉高風險，綠色代表拉低風險）。
* **【核心標題】**：**安全防禦 ｜ 博弈論特徵量化歸因**
* **【核心內容】**：
  * **可加性核心公式**：
    
    $$f(x) = \phi_0 + \sum_{i=1}^{M} \phi_i$$
    
  * **Shapley 邊際貢獻公式**：
    
    $$\phi_i(v) = \sum_{S \subseteq F \setminus \{i\}} \frac{|S|! (|F| - |S| - 1)!}{|F|!} [v(S \cup \{i\}) - v(S)]$$
    
  * **技術特點**：在全體特徵子集排列組合聯盟 $S$ 上進行遍歷遍歷，精算特徵 $i$ 參與前後的邊際預測提升度，給出無偏、唯一的公平分配。
* **【主講人】**：陳文家策略長
* **【時間管控】**：180 秒 (累計 18 分鐘)
* **【講師逐字稿】**：
  「這就是我們寫入 [單元五教材 (curriculum_v2026.md) 中的數學核心：SHAP 歸因公式](file:///g:/%E6%88%91%E7%9A%84%E9%9B%B2%E7%AB%AF%E7%A1%AC%E7%A2%9F/AI_Talent/curriculum/unit_5_explainable_ai/curriculum_v2026.md)。

  很多工程師寫報告時會用簡單的特徵重要性 (Feature Importance)，但那只能解釋『大盤大體上哪些特徵重要』，無法解釋『針對這單個客戶、這單個設備，AI 為什麼做出這個決定』。

  SHAP 完美的解決了這個問題。它基於合作博弈論理論。它把模型的預測輸出 $f(x)$，拆解為全廠平均值 $\phi_0$ 與各特徵 Shapley 值 $\phi_i$ 的相加。

  如果大盤平均良率是 92.5%，而這台機台預估良率只有 74.2%，我們的公式會精確算出來：是因為機台震動貢獻了 $-12.0\%$ 的拉低效應，冷卻液溫度貢獻了 $-8.5\%$。

  這在博弈論上是被數學證明是唯一、無偏的公平分配，**是目前全球金融與工業稽核上唯一被接受的『法定審計證據』。**」
