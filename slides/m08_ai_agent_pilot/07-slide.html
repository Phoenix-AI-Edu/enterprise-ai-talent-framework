安全防線二 ｜ TCO 財務熔斷控制

第二道安全關卡：Agentic Loop Guard 熔斷機制與 Max Iterations

$$\text{Total Cost} \le \text{Max Cost (\$2.00)} \quad \land \quad \text{Iterations} \le \text{Max Iterations (5)}$$
LIMITS

預算與重試限制

• Max Iterations = 5：限制單次任務工具重試上限為 5 次
• Max Cost = US$ 2.00：單筆對話 API 消耗上限，防止意外死循環耗盡預算

「這是我們專門為 CFO 設計的第二道安全防線：Loop Guard 熔斷機制。這也是我們輔導時，CFO 最感激的一項設計。

在普通的 RAG 或 Chatbot 中，使用者問一句，大模型答一句，Token 消耗是線性的。但 Agent 不一樣。Agent 是『自主運行』的。

當它在幫您跑一段財務報表分析時，如果資料庫的某個欄位格式不對，大模型會非常負責任地自我修正：*『啊，SQL 查詢出錯了，我換個查詢語句重新試一次！』*

如果沒有防護，它會在 1 秒鐘內嘗試幾十次，在幾分鐘內呼叫上萬次外部 API，產生無限循環。等您第二天上班，會看到一張驚人的數萬美元 API 帳單。

我們在程式底層配置了 Max Iterations 與 Max Cost 限制。一旦單次對話重試超過 5 次、或成本超過 2 美元，系統會強制關閉 API 連線，直接熔斷。這就叫代碼級的預算控制。」
