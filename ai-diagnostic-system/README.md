# AI Diagnostic System
Working materials for the **diagnostic pilot product** at Phoenix AI Consulting.

## Structure
- `docs/` — strategies, specs, positioning, research, and long-term reference
- `commercial/` — sales-facing pricing, packets, outreach
- `tests/` — anonymized problem sets and validation protocols
- `admin/` — SOW, legal, contracts, sign-off checklists

## Naming Convention
- Use `kebab-case` for filenames.
- If versioning is needed: `v{N}-{topic}`.

## Privacy Policy
- No client names, brands, logos, or directly identifying fields in `tests/` and `docs/`.
- Anonymized case assets: prefix with `ANON-` if stored here.

## Lifecycle
- Externally shared artifacts: `draft/` -> `review/` -> `published/`.
- Superseded research moves to `docs/archive/`.
