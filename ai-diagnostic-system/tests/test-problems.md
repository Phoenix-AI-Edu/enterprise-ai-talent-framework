**Context:** Anonymized, real-scope problems for alpha testing the diagnostic system.  
**Goal:** Validate intake completeness, retrieval quality, hypothesis rigor, consultant editability, client usability, and asset reusability per problem.

## Measurement Protocol

- **Baseline run:** Run all 10 against alpha system without consultant edits. Record outputs and time.
- **Consultant edit run:** Apply standardized edits; record per-problem edit time.
- **Client validation:** After each delivery, capture whether the client used output in an internal meeting.
- **Asset write-back:** After delivery, determine whether the problem generated >=1 reusable framework or pattern.
- **Go/No-go checkpoint:** After 6 problems, review whether the 3 success criteria can be met on pace.
